Fichier tsconfig.json


Le fichier tsconfig.json est un fichier de configuration essentiel dans les projets TypeScript et Angular. Il définit les options du compilateur TypeScript et les paramètres spécifiques à Angular.

Structure et Contenu


code :
voir code complet link 
            {
            "compileOnSave": false,
            "compilerOptions": {
                "outDir": "./dist/out-tsc",
                "strict": true,
                ...
                }
            }
  • "compileOnSave" : Détermine si le projet doit être recompilé automatiquement lors de la sauvegarde des fichiers. Dans Angular, cette option est généralement désactivée (false) car la compilation est gérée par le CLI Angular.

code :
voir code complet link 
                {
                "compilerOptions": {
                    "outDir": "./dist/out-tsc",
                    "strict": true,
                    "noImplicitOverride": true,
                    "noPropertyAccessFromIndexSignature": true,
                    "noImplicitReturns": true,
                    "noFallthroughCasesInSwitch": true,
                    "skipLibCheck": true,
                    "isolatedModules": true,
                    "esModuleInterop": true
                    }
                }
  • "outDir" : Spécifie le répertoire de sortie pour les fichiers JavaScript compilés.
  • "strict" : Active toutes les options de vérification stricte de TypeScript pour une meilleure sécurité de type.
  • "noImplicitOverride" : Exige que le mot-clé 'override' soit utilisé lors de la surcharge des méthodes.
  • "noPropertyAccessFromIndexSignature" : Interdit l'accès aux propriétés via la notation par points pour les types avec des signatures d'index.
  • "noImplicitReturns" : Assure que toutes les branches de code dans une fonction retournent une valeur.
  • "noFallthroughCasesInSwitch" : Signale les erreurs pour les cas de switch qui passent à travers sans break.
  • "skipLibCheck" : Ignore la vérification des fichiers de déclaration (*.d.ts) pour améliorer la vitesse de compilation.
  • "isolatedModules" : Assure que chaque fichier peut être compilé indépendamment.
  • "esModuleInterop" : Permet une meilleure interopérabilité entre les modules CommonJS et ES6.

code :
voir code complet link 
                {
                "compilerOptions": {
                    "experimentalDecorators": true,
                    "moduleResolution": "bundler",
                    "importHelpers": true,
                    "target": "ES2022",
                    "module": "ES2022"
                    }
                }
  • "experimentalDecorators" : Active le support des décorateurs, une fonctionnalité essentielle pour Angular.
  • "moduleResolution" : Définit comment TypeScript résout les imports. "bundler" est optimisé pour les bundlers modernes.
  • "importHelpers" : Permet l'importation des helpers tslib pour réduire la taille du code généré.
  • "target" : Spécifie la version d'ECMAScript pour le code JavaScript généré.
  • "module" : Définit le système de modules pour le code généré (ES2022 pour les modules ES6 modernes).

                    {
                "angularCompilerOptions": {
                    "enableI18nLegacyMessageIdFormat": false,
                    "strictInjectionParameters": true,
                    "strictInputAccessModifiers": true,
                    "strictTemplates": true
                    }
                }
  • "enableI18nLegacyMessageIdFormat" : Désactive le format legacy des IDs de messages pour l'internationalisation.
  • "strictInjectionParameters" : Active la vérification stricte des paramètres d'injection de dépendances.
  • "strictInputAccessModifiers" : Applique des vérifications strictes sur les modificateurs d'accès des entrées des composants.
  • "strictTemplates" : Active la vérification stricte des templates Angular pour une meilleure détection des erreurs.
